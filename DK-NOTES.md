# Using the forked package

Luckily, the vuejs/repl package was configured to use pr.new via github actions.
I've enabled that on our fork.

After the github actions have run, you can see the pr.new hash in the github actions logs:

https://github.com/oodavid/repl/actions

We've noticed that the URL generated points to the original repo, so we need to change it to point to our fork.

- This is the URL format generated by github actions:
  `npm i https://pkg.pr.new/@vue/repl@878a9b3`
- This is the URL we need to use to install the package from our fork:
  `npm i https://pkg.pr.new/oodavid/repl/@vue/repl@878a9b3`
